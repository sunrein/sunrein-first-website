Ruby Hashes vs JavaScript Objects - 11/15/2015
Differences in the Data Structures

Hashes in Ruby and objects in JavaScript are very similar. In fact, I found it helpful to think of them synonymously when starting JavaScript, having a background in Ruby, to figure out how objects work. However, they aren't quite the same, and from what I can tell, the main differences are found in syntax.

Hashes

Ruby hashes are data structures. They store information in pairs with a key and a value. Using the key when accessing information in a hash, we can access the value associated with it. In this way, a key can be thought of in similar terms to a variable assigned to a value. Keys and values can be any type of object, whether they be integers, strings, variables, and symbols. While "hash rockets" (=>) can be used in hashes when calling a value from a key, colon notation is more common. It should be noted that the keys with colons after them are symbols in Ruby. Here's an example below: my_info = { name: "Sabrina" age: 22 hometown: "Denver" }

Objects

Javascript objects are also data structures. In fact, they also look quite similar in terms of syntax, storing information in name/value pairs. Here's an example, using the same data I used above, just in JavaScript rather than Ruby. var my_info = { name: "Sabrina", age: 22, hometown: "Denver" };

As you can see above, there are a few differences. First, you have to declare the object as a variable using "var". If you do not do this, JavaScript won't be able to read the object. Also, there is more puncuation involved. For instance, good practicees for JavaScript dictate that commas should appear after each value (some Ruby style guides recommend you do this wish hashes as well, but I have not used them before). Also, there needs to be a semi-colon after the closing curly brace, just so JavaScript recognizes this as the end of the code block. Also, since there is no hash rocket alternative in JavaScript, the names with colons in JavaScript are not symbols. Also, names, unlike keys, can only be strings or variables. Values, just like values in hashes, can be integers, strings, and symbols.

Calling information from these structures

The other main difference in these structures stems from how you access the values stored within. In Ruby, if I wanted to call the value contained in the key name, it would look like this: my_info[:name] #this would return "Sabrina" If I wanted to access the value from the my_info object in JavaScript, the notation would look like this: my_info.name #this would return "Sabrina"

Conclusion

These are the fundamental differences in Ruby hashes and JavaScript objects. As you can see, the main differences are found in syntax and notation, but in theory, they function quite similarly. I'm sure there are some more subtle differences between the two that can be found through more practice and use of each structure in their respective languages.
